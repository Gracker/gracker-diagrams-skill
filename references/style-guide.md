# Gracker 风格指南

## 一句话定义

**用论文 visual abstract 的信息压缩能力，画出白板草图精修版的技术关系图。**

## 风格标签

- Excalidraw-like hand-drawn architecture
- visual abstract for systems
- 白板草图的精修版
- 极简技术关系图
- 浅色分类边框 + 大图标 + 无标题或弱标题

## 美感硬约束（所有图的默认法则）

以下规则**默认生效**，不需要在每个 prompt 里重复强调：

### 1. 内容居中 + 对称留白
- 内容必须在画布内**垂直居中**（上下等距）
- 水平居中（左右等距）
- 内容占画布约 70%，四周留白约 30%
- 不允许内容偏上偏下偏左偏右

### 2. 内容大小要匹配
- 内容不能太小（那是浪费画布）
- 内容也不能太大（失去呼吸感）
- 参考值：主体内容占据画布 65-75%

### 3. 留白是设计元素，不是空白
- 留白是美感的一部分
- 禁止把内容塞满画布四边
- 禁止内容偏在某一侧

### 4. 垂直居中是默认要求
- 内容偏上、偏下都是错误
- 内容块应该在画布内垂直居中

## 视觉硬约束

### 1. 背景
- **纯白底优先**，默认就是白色，不要偏黄纸感
- 只有在用户明确要求纸张感时，才退到极浅米白（接近 `#FAFAF7`）
- 不要复杂纹理，不要摄影背景
- 不要明显的奶油色、牛皮纸色、旧纸色

### 3. 线条
- 粗黑描边
- 允许轻微不规则，像手画但不凌乱
- 轮廓优先用圆角矩形
- 箭头尽量横平竖直，少用斜飞线

### 4. 颜色
只做分类，不抢信息。

推荐逻辑：
- 核心流程：浅橙
- 权限/规则：浅黄
- 工具/接口：浅蓝
- 状态/存储：浅绿
- 外部执行环境：浅灰

约束：
- 低饱和
- 高明度
- **优先用在模块外框、外圈 glow、顶部细边，不用大面积彩色底块**
- 主体仍以黑白信息为主

### 5. 图标
- 黑白简化图标
- 类似白板手绘或论文 visual abstract 图标
- **图标是主元素，不是点缀**
- 每个核心模块优先有一个大图标或清晰视觉符号
- 不要 3D 图标，不要 glossy 图标，不要表情化图标

### 6. 字体感觉（中英文同源原则）
- **中英文必须同一字体家族**，不允许分开选两套不同的字体
- 推荐组合（按优先级）：
  1. **LXGW WenKai（霞鹜文楷）**用于中英文——开源，手写感，稳，不幼态
  2. **Noto Sans CJK SC + Noto Sans**——中英文同属 Noto 家族，骨架天然对齐
  3. **Source Han Serif（思源宋体）+ Crimson Pro**——偏论文/书籍排版气质
- 正文字体以可读性优先，宁可少一点手写感，也不要飘、花、幼态
- 禁止：中文用一套字体、英文用另一套、两套骨架和笔触完全不搭

### 7. 标题处理
- **图自己是主角**，标题不是必须的
- 默认不单独做标题区
- 如果要标题：字体/大小/风格必须和所有标签完全一致，不允许有两套字体系统
- 位置要小、轻，可以放在左上角小标签或融入核心模块，不能占 1/3 空间
- 更推荐：直接用模块本身传达主题，不要单独醒目的标题区块

### 8. 注释标签 vs 流程节点
- **模块盒子和注释标签是两码事**，不能混淆
- 模块盒子：有边框、有背景色、有图标，是流程/关系图的主体
- 注释/标签：只有文字，无边框、无背景，是补充说明
- 常见错误：给 "LLM 召回"、"为什么两条RPC" 这类注释加模块边框，让它们看起来像流程节点
- 约束：注释标签只能是 clean text，或最多带一根引线指向目标，禁止给注释加流程节点式边框

### 9. 信息组织
- 优先画**关系**，不是优先写说明
- 优先做**中心系统 + 周边依赖** 或 **最短路径流程**
- 主体必须有视觉中心
- 周边模块围绕中心展开
- 一眼就能看出主路径
- 文案短，标签化，不写长段落
- **模块内默认不放 bullet list，除非信息压不进去**
- 留白必须够

## Perfetto 风格注入点

Perfetto 不等于深色 UI。这里取的是它的 **时间轨道 / 并行轨 / 阶段条** 语言。

适合注入到这些场景：
- 启动阶段
- 并发任务
- 请求-响应过程
- 压缩/恢复/重试链路
- 生命周期

做法：
- 在图下半区或侧边加横向 tracks
- 每条 track 用不同浅色块表示阶段
- 用短标签标注 duration / phase / wait / overlap
- 不要喧宾夺主，只作为技术细节层

## 禁止项

- corporate PPT 模板风
- 大面积渐变
- 玻璃拟态
- 高饱和 neon
- 写实插画
- 萌系卡通
- 过度装饰贴纸

## 判定标准

如果一张图同时满足下面 6 条，基本就对了：
1. 第一眼像技术白板，不像销售 PPT
2. 第二眼有层次，不乱
3. 读者能 3 秒内抓住主流程
4. 放进技术文章里不突兀，反而更清楚
5. 大图标和模块关系在主导视觉，不是标题或说明条在主导
6. 看起来更像 `01-main-structure.png`，而不是信息图海报
