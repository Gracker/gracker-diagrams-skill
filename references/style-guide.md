# Gracker 白板架构图风格指南

## 一句话定义

**Excalidraw 白板风格的系统关系图——像工程师在白板上画的那种，干净、直接、能放进技术博客。**

## 视觉锚点（最重要的部分）

这张图应该看起来像什么：

想象一个工程师站在会议室白板前，用黑色马克笔画系统架构。白板是标准办公白板（白色，可能略带灰调），线条是马克笔画的——粗、略有抖动、但清楚。模块是手画的矩形，用不同颜色的马克笔在边框上划了一道做分类。连接线是直线或直角折线，箭头是手画的。文字是工程师写的，清晰但不精致。

**不是什么：**
- 不是 iPad 手绘笔记（没有纸张质感、没有胶带、没有撕纸边缘）
- 不是 SaaS 产品 UI（没有圆角卡片、没有彩色顶边条、没有阴影）
- 不是插画/场景设计（没有桌面、没有显示器、没有马克杯）
- 不是水彩画（没有晕染、没有渐变、没有柔和过渡）
- 不是海报/信息图（没有标题区、没有装饰、没有金句）

## 美感硬约束

### 1. 背景
- **标准白色白板**，允许极微弱的灰调（`#FAFAFA` 到 `#F5F5F5`）
- **绝对禁止**：奶油色、牛皮纸色、纸质纹理、暖色调
- 白板就是白的，不需要"有质感"

### 2. 线条
- **粗黑马克笔线条**，像白板笔画的
- 允许自然的手抖，但不夸张
- 线宽一致（不像真正手画那样粗细不均）
- 轮廓用矩形，允许轻微圆角（但不是 UI 级别的圆角）
- **连接线必须直角折线**，禁止斜线飞线
- 箭头简单：两条短线拼成 V 形，不要三角实心箭头

### 3. 颜色（极度克制）
- **主体是黑白**：白色背景 + 黑色线条 + 黑色文字
- **颜色只做一件事：区分分类**
- 用法：模块边框用单色马克笔描一道（仅边框，不填充）
- 推荐配色（低饱和，像白板笔的颜色）：
  - 核心流程：深蓝 `#4A90D9`
  - 权限/规则：深橙 `#E8913A`
  - 工具/接口：青色 `#5BA4CF`
  - 状态/存储：深绿 `#4CAF50`
  - 外部系统：灰色 `#9E9E9E`
- **颜色数量**：一张图最多 4 种颜色，超过就乱了
- **绝对禁止**：浅色填充、彩色背景块、渐变、大面积色块

### 4. 图标
- **极简线条图标**，像工程师在白板上画的简笔画
- 每个核心模块可以有一个小图标
- 图标是线条画，不是填充画
- 图标大小一致，不作为主视觉元素（模块和关系才是主视觉）
- 禁止 3D 图标、禁止 emoji 风格、禁止精细插画

### 5. 文字
- 清晰可读，像工程师手写体
- 标题和标签用同一风格，不做大小/粗细层级区分（白板上不会换笔）
- **中英文同源字体**：推荐 LXGW WenKai 或 Noto Sans CJK
- 模块内文字短，标签化，不写段落
- 不做独立标题区——图本身说明一切

### 6. 信息密度
- **密度由内容决定**：简单内容模块大留白多，复杂内容紧凑
- 核心原则：**能放进技术博客文章里的密度**——读者在手机上也能看清
- 模块内默认不放 bullet list，除非信息压不进去

### 7. 空间
- 内容垂直 + 水平居中
- 内容占画布 60-70%
- 留白均匀，不偏不挤
- 模块之间间距一致

## Perfetto 风格注入点

与之前一致——只在内容有真实时间维度时才加横向 tracks。
用白板笔风格画：细长矩形条，不同颜色区分阶段，标签写在旁边。
不要喧宾夺主。

## 禁止项（完整列表）

### 绝对禁止
- 奶油色 / 暖色纸张背景
- 纸张纹理、胶带、撕纸边缘
- 圆角卡片 + 彩色顶边条（SaaS UI 风）
- 阴影、投影、深度效果
- 渐变（任何形式）
- 水彩晕染、柔和过渡
- 场景插画（桌面、显示器、人物）
- 3D 元素
- emoji 风格图标
- 精细插画图标
- 大面积彩色填充
- 独立标题区
- 装饰性元素（星星、贴纸、花纹）
- 高饱和 neon 色
- 玻璃拟态
- corporate PPT 模板
- 手绘波浪箭头（那是马卡龙信息图的）

### 默认不使用但内容需要时可以
- Perfetto-style timeline tracks（有时序数据时）
- 虚线框分组（模块关系清晰时）
- 简短注释标签（无边框，只有文字 + 引线）

## 判定标准

一张图同时满足以下 6 条 = 对了：

1. **第一眼像白板照片**，不像 PPT、不像插画、不像 UI
2. **颜色极少**，黑白为主，最多 4 种马克笔色做分类
3. **线条是马克笔风格**——粗、直、略有抖动
4. **模块是白板上的矩形**——边框描色，内部白色，没有填充
5. **3 秒内能抓住主流程/主关系**
6. **放进技术博客文章里自然**，不突兀，不抢正文风头
