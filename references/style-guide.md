# Gracker 风格指南

## 一句话定义

**用手绘白板的亲和力，画出论文级的信息组织。**

## 风格标签

- Excalidraw-like hand-drawn architecture
- visual abstract for systems
- 白板草图的精修版
- 技术手账式架构图

## 视觉硬约束

### 1. 背景
- 白底优先
- 或浅米色纸张底（接近 `#FFF8F0`）
- 不要复杂纹理，不要摄影背景

### 2. 线条
- 粗黑描边
- 允许轻微不规则，像手画但不凌乱
- 轮廓优先用圆角矩形
- 箭头尽量横平竖直，少用斜飞线

### 3. 颜色
只做分类，不抢信息。

推荐逻辑：
- 核心流程：浅橙
- 权限/规则：浅黄
- 工具/接口：浅蓝
- 状态/存储：浅绿
- 外部执行环境：浅灰

约束：
- 低饱和
- 高明度
- 面积小，更多用于边框或局部高亮
- 主体仍以黑白信息为主

### 4. 图标
- 黑白简化图标
- 类似白板手绘或论文 visual abstract 图标
- 不要 3D 图标，不要 glossy 图标，不要表情化图标

### 5. 字体感觉
- 手写感或接近手写的亲和字体
- 但必须清楚、稳，不要儿童化
- 标题可以更有手感，正文标签保持可读性

### 6. 信息组织
- 主体必须有视觉中心
- 周边模块围绕中心展开
- 一眼就能看出主路径
- 文案短，标签化，不写长段落
- 留白必须够

## Perfetto 风格注入点

Perfetto 不等于深色 UI。这里取的是它的 **时间轨道 / 并行轨 / 阶段条** 语言。

适合注入到这些场景：
- 启动阶段
- 并发任务
- 请求-响应过程
- 压缩/恢复/重试链路
- 生命周期

做法：
- 在图下半区或侧边加横向 tracks
- 每条 track 用不同浅色块表示阶段
- 用短标签标注 duration / phase / wait / overlap
- 不要喧宾夺主，只作为技术细节层

## 禁止项

- corporate PPT 模板风
- 大面积渐变
- 玻璃拟态
- 高饱和 neon
- 写实插画
- 萌系卡通
- 过度装饰贴纸

## 判定标准

如果一张图同时满足下面 4 条，基本就对了：
1. 第一眼像技术白板，不像销售 PPT
2. 第二眼有层次，不乱
3. 读者能 3 秒内抓住主流程
4. 放进技术文章里不突兀，反而更清楚
