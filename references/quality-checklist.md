# 质量检查清单

生成图片后，用以下清单验证。

## 通过标准（必须全部满足）

- [ ] 第一眼像技术白板，不像销售 PPT
- [ ] 第二眼有层次，不乱
- [ ] 读者能 3 秒内抓住主流程
- [ ] 放进技术文章里不突兀
- [ ] 线条是手绘感，不是完美机械描边
- [ ] 颜色是低饱和浅色，不是高饱和
- [ ] 背景明显偏白，不带明显纸张黄底
- [ ] 字体统一、稳定、可读，不飘不花
- [ ] **内容垂直居中**，上下留白对称，不偏上不偏下
- [ ] **内容水平居中**，左右留白对称，不偏左不偏右
- [ ] **内容大小合适**，不撑满也不太小（参考：内容占画布 65-75%）
- [ ] **中英文字体同源**，同一家族，不混搭两套

## Perfetto track 检查（如果图里有时序/阶段）

- [ ] 横向 timeline/tracks 存在
- [ ] 颜色与主图配色体系一致
- [ ] 阶段标注清晰
- [ ] 不抢主图视线

## 禁止出现

- [ ] 大面积渐变
- [ ] 玻璃拟态
- [ ] 高饱和 neon
- [ ] 写实插画
- [ ] 萌系卡通
- [ ] corporate PPT 模板感
- [ ] 过度装饰贴纸
- [ ] 信息过密导致无法阅读
- [ ] 内容偏上/偏下/偏左/偏右（留白不对称）

## 不满足时的处理

1. 记录哪个检查项失败
2. 针对性改 prompt（只改失败项）
3. 重试一次
4. 还不行就换个布局方向重新来

不要无止境重抽。最多 2 轮。
