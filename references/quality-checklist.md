# 质量检查清单

生成图片后，用以下清单验证。

## 通过标准（必须全部满足）

### 白板感（核心）
- [ ] 第一眼像白板照片，不像 PPT/UI/插画
- [ ] 白色背景，没有奶油色/纸张纹理/暖色调
- [ ] 线条是马克笔风格——粗、直、略有抖动
- [ ] 模块是矩形边框描色，内部白色，没有填充
- [ ] 箭头是手画 V 形，不是三角实心

### 颜色
- [ ] 黑白为主，彩色仅做分类
- [ ] 最多 4 种颜色
- [ ] 没有浅色填充、彩色顶边条、大面积色块
- [ ] 没有渐变、没有高饱和 neon

### 图标
- [ ] 极简线条图标，像白板简笔画
- [ ] 没有 emoji 风格、没有精细插画、没有 3D

### 信息
- [ ] 3 秒内能抓住主流程/主关系
- [ ] 文字清晰可读，标签化，不写段落
- [ ] 放进技术博客里不突兀

### 空间
- [ ] 内容垂直 + 水平居中
- [ ] 留白均匀，不偏不挤
- [ ] 内容占画布 60-70%

### 排除项（出现任何一项 = 不合格）
- [ ] 奶油色/暖色纸张背景
- [ ] 纸张纹理、胶带、撕纸边缘
- [ ] 圆角卡片 + 彩色顶边条（SaaS UI 风）
- [ ] 阴影、投影
- [ ] 渐变
- [ ] 水彩晕染
- [ ] 场景插画（桌面、显示器、人物）
- [ ] 大面积彩色填充
- [ ] 独立标题区
- [ ] 装饰元素（星星、贴纸、花纹）
- [ ] 手绘波浪箭头（那是马卡龙信息图的特征）

## Perfetto track 检查（如果图里有时序/阶段）

- [ ] 横向 tracks 存在
- [ ] 颜色与主图配色一致
- [ ] 画法是简单细长条，不是精细 UI
- [ ] 不抢主图视线

## 不满足时的处理

1. 记录哪个检查项失败
2. 针对性改 prompt（只改失败项）
3. 重试一次
4. 还不行就换个布局方向重新来

不要无止境重抽。最多 2 轮。
