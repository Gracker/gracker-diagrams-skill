# 质量检查清单

生成图片后，用以下清单验证。

## 通过标准（必须全部满足）

- [ ] 第一眼像技术白板，不像销售 PPT
- [ ] 第二眼有层次，不乱
- [ ] 读者能 3 秒内抓住主流程
- [ ] 放进技术文章里不突兀
- [ ] 线条是手绘感，不是完美机械描边
- [ ] 颜色是低饱和浅色，不是高饱和

## Perfetto track 检查（如果图里有时序/阶段）

- [ ] 横向 timeline/tracks 存在
- [ ] 颜色与主图配色体系一致
- [ ] 阶段标注清晰
- [ ] 不抢主图视线

## 禁止出现

- [ ] 大面积渐变
- [ ] 玻璃拟态
- [ ] 高饱和 neon
- [ ] 写实插画
- [ ] 萌系卡通
- [ ] corporate PPT 模板感
- [ ] 过度装饰贴纸
- [ ] 信息过密导致无法阅读

## 不满足时的处理

1. 记录哪个检查项失败
2. 针对性改 prompt（只改失败项）
3. 重试一次
4. 还不行就换个布局方向重新来

不要无止境重抽。最多 2 轮。
